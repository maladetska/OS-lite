МОДИФИКАЦИЯ:

Продемонстрируйте умение работать с утилитой top в интерактивном режиме. 
1) Выведите только те процессы, которые запущены вашим юзером.
2) По какому принципу упорядочены процессы в top? Отсортируйте данные по объёму занимаемой памяти.
3) Как часто top обновляет информацию? Попробуйте изменить время обновления данных таблицы.
4) Выведите только активные (работающие) на текущий момент процессы.
5) Выясните, как можно убить процесс по заданному PID (для безопасности не следует убивать процесс, просто расскажите алгоритм).

РЕАЛИЗАЦИЯ:

Команды: - 
Комментарии: *()

*(Для выполнения всех задач сначала вводим в терминале команду top)

1) Выведите только те процессы, которые запущены вашим юзером.

-> нажимаем клавишу u или сочетание клавиш shift+u
*(появится строка над таблицей: Which user (blank for all)) 
-> пишем туда имя текущего пользователя
готово!

2) По какому принципу упорядочены процессы в top? Отсортируйте данные по объёму занимаемой памяти.

*(Изначально таблица сортируется по столбцу %CPU. Поэтому надо переместиться на четыре колонки влево, чтобы попасть на VIRT или на одну -- на  %MEM)
-- VIRT: для перемещения влево используем сочетание клавиш Shift + < , где < - это клавиша (Б, <), сделать четыре раза
-- %MEM: для перемещения влево используем сочетание клавиш Shift + > , где > - это клавиша (Ю, <), сделать один раз раза
- таким образом, если мы хотим переключаться по столбцам для их сортировки надо использовать Shift + < или Shift + >
готово!

3) Как часто top обновляет информацию? Попробуйте изменить время обновления данных таблицы.

- нажимаем клавишу d или s
*(Над таблицей появится надпись "Change delay from 3,0 to ", значит top автоматически обновляется раз в три секунды)
- вводим число целое или обыкновенную дробь, чтобы изменить время обновления данных таблицы
*(лучше не вводить 0, иначе программа будет непрерывно обновлять информацию и при этом потреблять много процессорного времени)
готово!

4) Выведите только активные (работающие) на текущий момент процессы.

- нажимаем клавишу i, ею можно переключаться между всеми или только активными процессами, 
- если "активными" считаются те процессы, которые работают в данный момент, то есть имеют статус RUNNING, тогда повторяем до момента, когда в колонке S не будут все R 
готово!

5) Выясните, как можно убить процесс по заданному PID (для безопасности не следует убивать процесс, просто расскажите алгоритм).

- нажимаем клавишу k
*(появится строка над таблицей: PID to signal/kill [default pid = 1521])
